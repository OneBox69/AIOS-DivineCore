# Branding OS — Module 1 (Creative Intelligence)

Highest strategic value module. Full content pipeline from idea → publish-ready output. Five named agents, each with one job.

## Status

- **IMAGYN** — active (idea generation)
- **LYRA, MONTAGE, FORGE, VIGIL** — in build / scaffold

Local folders are mostly `.gitkeep` placeholders today. Real architecture is documented in CLAUDE.md §6 Module 1.

## Agents

| Agent | Role | KB source |
|-------|------|-----------|
| IMAGYN | Idea generation | Kallaway KB (Airtable) + working examples (Supabase) |
| LYRA | Script writing | Kallaway KB (emotional resonance, story structure) |
| MONTAGE | Editing brief | Pacing/b-roll/music conventions |
| FORGE | Visual prompt generation | Diffusion model best practices (Midjourney/Runway/Flux) |
| VIGIL | Performance feedback | Instagram Graph API + YouTube Analytics |

## Knowledge base architecture

| Layer | Tool | Purpose |
|-------|------|---------|
| **Core KB** | Airtable | Kallaway's lessons. 51+ entries. Exact retrieval. Fields: Title, Category, Core Lesson, Full Explanation, Content Attributes, Content Types, Source URL, Notes |
| **Working examples** | Supabase + pgvector | Real high-performing content. Embedded via OpenAI text-embedding-3-small. Semantic search by meaning, not keyword. |

## Context sources (REQUIRED — every Branding OS agent loads these)

Branding OS produces first-person, brand-voiced content. Every agent here MUST pull identity context from `shared/context/` in addition to its domain KB:

| File | Why it matters for Branding OS |
|------|-------------------------------|
| [`shared/context/identity/voice.md`](../shared/context/identity/voice.md) | Brand-level tone, banned phrases, per-channel formatting. The single source of truth for *how* anything sounds. |
| [`shared/context/identity/mayank.md`](../shared/context/identity/mayank.md) | Mayank's POV — video content is first-person. Loads worldview, signature lines, things he says often. |
| [`shared/context/identity/strategy.md`](../shared/context/identity/strategy.md) | Current campaigns + positioning. Prevents agents from writing yesterday's pitch. |
| [`shared/context/identity/audience.md`](../shared/context/identity/audience.md) | Reader profile — operator-mindset, time-poor, skeptical of AI hype. Determines hook style and example choice. |

For **Pang's content** (LinkedIn primarily) load [`shared/context/identity/pang.md`](../shared/context/identity/pang.md) instead of `mayank.md`, plus [`shared/context/playbooks/linkedin-playbook.md`](../shared/context/playbooks/linkedin-playbook.md) for LinkedIn-specific tactics.

Pattern: an agent's system prompt should explicitly cite the relevant `shared/context/` files (e.g. *"Use the tone described in `shared/context/identity/voice.md`"*) so drift is visible and correctable.

## Cost philosophy

Where API calls to generation models (video, image) are prohibitively expensive, FORGE outputs the **prompt** and a human executes manually. Ops OS alerts the human when a prompt is ready and time-sensitive.

## Targets

- Platforms: YouTube + Instagram (primary), LinkedIn + Reddit (secondary)
- Cadence: minimum one piece of content per day

## What does NOT go here

- VIGIL's analytics ingestion infrastructure — that goes in `divinecore-v2/` if/when it becomes a service.
- The actual Airtable / Supabase data — only the *sync scripts* live here.
