# sales_os/integrations/upwork/

Two Celery tasks:
- `tasks.upwork_generate_proposal` — turns a raw Upwork job description into a customised application body, a Google Doc proposal, and a tracking-sheet row.
- `tasks.upwork_finalize_proposal` — patches the existing tracking-sheet row with `Connects used` (column E) and `Loom video link` (column I) once the user has applied. Triggered by `POST /upwork/finalize` from the result page form.

Originally migrated from a 3-workflow n8n system. The per-job sales-script Doc + Mermaid diagram step that existed in the n8n version was dropped — Pang uses one standardised sales script across all Upwork calls, so per-job script generation was dead weight (LLM cost + Drive clutter). The Loom video carries the AIOS framing instead; see the script in CLAUDE.md / sales-playbook.

## Flow

```
upwork_generate_proposal(job_description)
  -> llm.proposal_fields(jd)            # JSON: title/brief/steps/flow/about_me_bullets
  -> llm.application_copy(jd)           # application body, contains $$$ placeholder
  -> google_client.copy_doc(proposal_template) -> proposal_id
  -> google_client.share_doc(proposal_id, "reader")
  -> google_client.fill_doc(proposal_id, fields)
  -> row_index = google_client.append_row(tracking_sheet,
       [date, title, jd, brief, "", "", proposal_url, "", ""])
  -> return {application_body: body.replace("$$$", proposal_url), proposal_url, row_index}

upwork_finalize_proposal(row_index, base_connects, boosted_connects, loom_url)
  -> connects = "{base} + {boosted}" (or single value, or empty)
  -> google_client.update_cells(tracking_sheet, {"Sheet1!E{row}": connects, "Sheet1!I{row}": loom})
```

## Files

| File | Role |
|------|------|
| `processor.py` | Celery task entry points: `upwork_generate_proposal` and `upwork_finalize_proposal`. |
| `llm.py` | Two OpenRouter calls — `proposal_fields` (JSON) and `application_copy` (JSON). Same OpenAI-SDK-against-OpenRouter pattern as `fathom/categorizer.py`. |
| `google_client.py` | Drive/Docs/Sheets helpers. `copy_doc`, `share_doc`, `fill_doc` (batchUpdate replaceAllText), `append_row` (returns new row index), `update_cells` (batchUpdate by A1 ranges). Auth via stored OAuth refresh token. |
| `prompts.py` | Two prompt templates. Voice rules sourced from [shared/context/voice.md](../../../shared/context/voice.md). The application body prompt enforces the AIOS framing ("I don't build commoditized automations. I build AI Operating Systems — your business, running on AI...") and bans agency / co-founder framing for Upwork specifically. |
| `about_me.py` | `ABOUT_ME` string constant — Upwork-specific About Me, intentionally stripped of agency/co-founder framing. Loosely derived from [shared/context/pang.md](../../../shared/context/pang.md) but **not** an auto-mirror; edit directly. |
| `config.py` | Proposal template Doc ID, Sheet ID, placeholder names, column letters. Single place to change. Pulls from `worker/settings.py`. |
| `oauth_bootstrap.py` | One-shot script — run once to get a Google OAuth refresh token. See [Setup](#setup) below. |

## Setup (one-time, before first run)

1. **Google Cloud Console** — make sure you have an OAuth 2.0 Client ID. Reuse the one n8n uses if available. Type: Desktop / Web app. Add `http://localhost:8080/` as an authorised redirect URI. Enabled APIs: Docs, Drive, Sheets.
2. **Add to `.env`**:
   ```
   GOOGLE_OAUTH_CLIENT_ID=...
   GOOGLE_OAUTH_CLIENT_SECRET=...
   ```
3. **Run the bootstrap** from `divinecore-v2/`:
   ```
   docker compose run --rm worker python -m sales_os.integrations.upwork.oauth_bootstrap
   ```
4. Follow the prompts: open the URL, grant access, paste the redirected URL back. The script prints the refresh token.
5. **Add to `.env`**: `GOOGLE_OAUTH_REFRESH_TOKEN=...`. Done forever (until revoked).

## Doc placeholders

The pipeline uses `replaceAllText` to substitute `{{key}}` markers in the proposal Doc template. Markers must match exactly:

- **Proposal Doc**: `{{titleOfSystem}}`, `{{briefExplanationOfSystem}}`, `{{stepByStepBulletPoints}}`, `{{leftToRightFlowWithArrows}}`, `{{aboutMeBulletPoints}}`

If you change the template Doc layout, keep these placeholder names — or update both the template and `processor.py` together.

## Tracking sheet schema

Columns (left to right): `Date & time` (A), `Upwork application Title` (B), `Upwork application description (full)` (C), `Upwork application description` (D), `Connects used` (E), `Status` (F), `Google doc proposal sent` (G), `Google doc sales script used` (H), `Loom video link` (I).

- Generation pipeline (`upwork_generate_proposal`) writes A–D + G; leaves E, F, H, I blank.
- Finalize pipeline (`upwork_finalize_proposal`, triggered by the result-page form after applying) writes E (`Connects used` — formatted `"{base} + {boosted}"`, e.g. `"15 + 5"`) and I (`Loom video link`).
- Column F (`Status`) remains manual.
- Column H (`Google doc sales script used`) is now permanently empty since per-job sales-script generation was dropped. The header can be removed/repurposed in the sheet directly when convenient.

Column letters are configurable via `TRACKING_SHEET_CONNECTS_COLUMN` / `TRACKING_SHEET_LOOM_COLUMN` in `config.py` if the sheet layout shifts.

## Voice — do not break

About Me content lives in `about_me.py`. Banned phrases enforced in `prompts.py` (sourced from [shared/context/voice.md](../../../shared/context/voice.md)).

**Upwork-specific voice rules baked into the prompts:**
- No "co-founder" framing for Pang.
- No mention of DivineSide.
- No use of the word "agency" anywhere in output.
- Application body must lead with relevant past work claim ("I've built the exact workflow before…" / "I built something like this for another client…") followed by an industry tie if applicable.
- Application body must include the verbatim AIOS sentence: *"I don't build commoditized automations. I build AI Operating Systems — your business, running on AI."*

These rules deliberately diverge from the playbook's "never lead with AI Operating System externally" guidance ([shared/context/sales-playbook.md](../../../shared/context/sales-playbook.md) line 173). On Upwork specifically the AIOS phrase is the differentiator that justifies the price floor; on other channels (LinkedIn, sales calls) the playbook still applies.

## Open issues

- Pipeline has no retries on Google API transient errors. If a Doc copy fails, the user sees a 500. Add Celery `autoretry_for` / `retry_backoff` if this becomes annoying in practice.
- About Me bullets in the application body are picked by the LLM from `about_me.py`. If the LLM keeps surfacing the wrong case study for a given JD type, tighten the prompt's relevance rules.
