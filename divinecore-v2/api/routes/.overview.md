# api/routes/

Route modules for the v2 FastAPI service. Pattern: one file per integration / domain. Each file exports a `router = APIRouter(...)` that `api/main.py` mounts via `include_router`.

## Files

| File | Mount | Purpose |
|------|-------|---------|
| `fathom.py` | `POST /webhooks/fathom` | Receives Fathom meeting recording webhooks. Verifies HMAC-SHA256 signature against `FATHOM_WEBHOOK_SECRET`, then enqueues `tasks.process_fathom_recording` on Celery. Returns 401 on invalid signature, 200 with `{received, meeting_id}` otherwise. |

## Conventions

- Routers use a `prefix` (e.g. `/webhooks`) and `tags` so OpenAPI docs stay organised.
- HMAC secrets live in `api/settings.py` (Pydantic Settings). Never hardcode.
- Webhook handlers should be thin: verify -> enqueue -> return. All real work happens in the worker.
- Don't `await` Celery results inside a request handler — fire-and-forget via `send_task`.

## Adding a new route module

1. Create `<name>.py` exposing `router = APIRouter(...)`.
2. Import + `app.include_router(...)` in `api/main.py`.
3. Add new env vars to `api/settings.py`.
4. Update this file's table.
