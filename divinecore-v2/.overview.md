# divinecore-v2 — Code-First Runtime

The Python code-first runtime that runs in parallel to the n8n orchestration stack. Long-term, this replaces n8n for agent execution. Today, it's a foundation: API + worker + scheduler + broker, all containerised.

- **Foundation shipped** (commit `3c03553`)

## Structure

```
divinecore-v2/
├── api/
│   ├── main.py                ← FastAPI app, mounts routers
│   ├── settings.py            ← Pydantic Settings (REDIS_URL, FATHOM_WEBHOOK_SECRET)
│   ├── routes/
│   │   └── fathom.py          ← POST /webhooks/fathom (HMAC verify -> Celery)
│   ├── requirements.txt
│   └── Dockerfile
├── worker/
│   ├── celery_app.py          ← Celery config + beat_schedule + task includes
│   ├── tasks.py               ← echo, heartbeat (demo)
│   ├── settings.py            ← Pydantic Settings (Airtable, Anthropic, Redis)
│   ├── team.py                ← TEAM_MEMBERS dict + email/name lookup
│   ├── integrations/
│   │   └── fathom/            ← Meeting recorder integration (see its .overview.md)
│   ├── requirements.txt
│   └── Dockerfile
├── docker-compose.yml         ← redis + api + worker + beat
└── .dockerignore
```

## Services (docker-compose.yml)

| Service | Image / Build | Notes |
|---------|---------------|-------|
| `redis` | `redis:7-alpine`, port 6379 | Broker AND result backend for Celery |
| `api` | `./api` | FastAPI on `:8000`. Mounts `./api` for live reload in dev. |
| `worker` | `./worker` | Celery worker. Same image as `beat`. |
| `beat` | `./worker` (override CMD) | Celery Beat scheduler. Currently runs `tasks.heartbeat` every 30s. |

`REDIS_URL=redis://redis:6379/0` is passed to all three Python services.

## Endpoints (today)

- `GET /` — health
- `POST /tasks/echo` — submit a Celery task, returns `task_id`
- `GET /tasks/{task_id}` — poll status + result
- `POST /webhooks/fathom` — Fathom meeting webhook. HMAC-verified. Enqueues `tasks.process_fathom_recording`. See [worker/integrations/fathom/.overview.md](worker/integrations/fathom/.overview.md).

## Integrations

- **Fathom** — meeting recordings -> Airtable `Meetings` table + opt-in Pulse Task creation. See [worker/integrations/fathom/.overview.md](worker/integrations/fathom/.overview.md). Hosting / public webhook URL is handled outside this repo (user provides ingress).

## Run locally

```bash
cd divinecore-v2
docker compose up --build
```

## What goes here

- New API routes → `api/routes/<name>.py`, then `app.include_router` in `api/main.py`
- New API env vars → `api/settings.py`
- New worker integrations → `worker/integrations/<service>/`, add to `include` list in `celery_app.py`
- New worker env vars → `worker/settings.py`
- New scheduled jobs → add to `app.conf.beat_schedule` in `worker/celery_app.py`
- New service (e.g. a Discord listener) → new top-level folder + add to `docker-compose.yml`

## What does NOT go here

- n8n workflow JSONs — those go in the relevant module's `workflows/` folder.
- Agent system prompts — those go in the relevant module's `agents/` folder.
- Knowledge-base sync scripts — those go in the relevant module's `knowledge-base/` folder.
