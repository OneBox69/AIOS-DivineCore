# worker/integrations/upwork/

Single Celery task `tasks.upwork_generate_proposal` — turns a raw Upwork job description into a customised application body, a Google Doc proposal, a Google Doc sales script (with Mermaid flowchart), and a tracking-sheet row.

Migrated from a 3-workflow n8n system (`⭐⭐⭐Upwork AI Agent` orchestrator + `Generate google docs` + `Generate application copy`). The n8n version stays as fallback until v2 is proven.

## Flow

```
upwork_generate_proposal(job_description)
  -> llm.proposal_fields(jd)            # JSON: title/brief/steps/flow/about_me_bullets
  -> llm.mermaid(jd)                    # raw Mermaid flowchart code
  -> llm.application_copy(jd)           # short application body, contains $$$ placeholder
  -> google_client.copy_doc(proposal_template) -> proposal_id
  -> google_client.share_doc(proposal_id, "reader")
  -> google_client.fill_doc(proposal_id, fields)
  -> google_client.copy_doc(script_template) -> script_id
  -> google_client.share_doc(script_id, "writer")
  -> google_client.fill_doc(script_id, {summary, steps, mermaid})
  -> google_client.append_row(tracking_sheet, [date, title, jd, brief, "", "", proposal_url, script_url, ""])
  -> return {application_body: body.replace("$$$", proposal_url), proposal_url, script_url}
```

## Files

| File | Role |
|------|------|
| `processor.py` | Celery task entry point. Composes the 8-step pipeline. |
| `llm.py` | Three OpenRouter calls — `proposal_fields` (JSON), `mermaid` (raw text), `application_copy` (JSON). Same OpenAI-SDK-against-OpenRouter pattern as `fathom/categorizer.py`. |
| `google_client.py` | Drive/Docs/Sheets helpers. `copy_doc`, `share_doc`, `fill_doc` (batchUpdate replaceAllText), `append_row`. Auth via stored OAuth refresh token. |
| `prompts.py` | Three prompt templates. Voice rules sourced from [shared/context/voice.md](../../../../shared/context/voice.md). Doc body templates that get rendered (the `Hi {Name}…` text the Doc placeholders sit inside). |
| `about_me.py` | `ABOUT_ME` string constant — manual mirror of [shared/context/pang.md](../../../../shared/context/pang.md). When pang.md changes, sync here. |
| `config.py` | Template Doc IDs, Sheet ID, Doc placeholder names. Single place to change. Pulls from `worker/settings.py`. |
| `oauth_bootstrap.py` | One-shot script — run once to get a Google OAuth refresh token. See [Setup](#setup) below. |

## Setup (one-time, before first run)

1. **Google Cloud Console** — make sure you have an OAuth 2.0 Client ID. Reuse the one n8n uses if available. Type: Desktop / Web app. Add `http://localhost:8080/` as an authorised redirect URI. Enabled APIs: Docs, Drive, Sheets.
2. **Add to `.env`**:
   ```
   GOOGLE_OAUTH_CLIENT_ID=...
   GOOGLE_OAUTH_CLIENT_SECRET=...
   ```
3. **Run the bootstrap** from `divinecore-v2/`:
   ```
   docker compose run --rm worker python -m integrations.upwork.oauth_bootstrap
   ```
4. Follow the prompts: open the URL, grant access, paste the redirected URL back. The script prints the refresh token.
5. **Add to `.env`**: `GOOGLE_OAUTH_REFRESH_TOKEN=...`. Done forever (until revoked).

## Doc placeholders

The pipeline uses `replaceAllText` to substitute `{{key}}` markers in the template Docs. Markers must match exactly:

- **Proposal Doc**: `{{titleOfSystem}}`, `{{briefExplanationOfSystem}}`, `{{stepByStepBulletPoints}}`, `{{leftToRightFlowWithArrows}}`, `{{aboutMeBulletPoints}}`
- **Sales script Doc**: `{{shortSummary}}`, `{{stepByStepBuilding}}`, `{{mermaidCode}}`

If you change template Doc layouts, keep these placeholder names — or update both the template and `processor.py` together.

## Tracking sheet schema

Append-only. Columns (left to right): `Date & time`, `Upwork application Title`, `Upwork application description (full)`, `Upwork application description`, `Connects used`, `Status`, `Google doc proposal sent`, `Google doc sales script used`, `Loom video link`. The pipeline writes columns 1–4 + 7–8; columns 5, 6, 9 are filled manually post-application.

## Voice — do not break

About Me content lives in `about_me.py`, mirrored from [shared/context/pang.md](../../../../shared/context/pang.md). Banned phrases enforced in `prompts.py` (sourced from [shared/context/voice.md](../../../../shared/context/voice.md) lines 38–50). When voice.md or pang.md changes meaningfully, sync.

## Open issues

- Mermaid is embedded as raw code in the sales-script Doc, same as the n8n version. Render-to-image is a follow-up if you want it visual in the proposal.
- Pipeline has no retries on Google API transient errors. If a Doc copy fails, the user sees a 500. Add Celery `autoretry_for` / `retry_backoff` if this becomes annoying in practice.
