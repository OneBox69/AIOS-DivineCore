# shared/context/sales-and-delivery — Sales & Delivery

Everything from "they replied to outreach" through to "the pilot workflow is live and they're on retainer." One unified offer across all channels (cold email, Upwork, LinkedIn, inbound), with `offer.md` as the strategic layer and `guarantee.md` as the legal-mechanic deep-dive.

## Files

| File | What it contains |
|------|------------------|
| `offer.md` | **Unified canonical offer across all channels.** Headline promise ("Your business, running on AI."), naming discipline (outcome-first by default, AIOS exception on Upwork), pilot phases (8 weeks: context → data → build → verify), pricing anchors (~£2.5k pilot / ~£2.5k retainer, provisional), payment model (0% upfront, 100% on delivery), scope envelope, expansion path, channel/audience map. |
| `guarantee.md` | **Guarantee mechanic deep-dive.** TYPE A (hours-saved) vs TYPE B (functions-delivered, default), the four-signal test for picking TYPE A, verbatim guarantee paragraphs as they appear in the application body and proposal Doc, what triggers / doesn't trigger, buyer obligations, voids, claim mechanic. Pairs with `offer.md`. |
| `sales-playbook.md` | End-to-end sales SOP — qualification, pre-call audit checklist, 24h audit doc template, follow-up cadence, pilot→retainer transition. The future Sales OS module loads from this. |
| `sales-discovery-call.md` | 45-min discovery call playbook — call structure, scripts, decision-authority branches, why-triad diagnosis, offer framing with temperature checks, risk reversal, objection responses. Load before any live call. |
| `intake-form.md` | Client intake form — 10-question Typeform sent in Phase 0. Captures easy facts async; voice/strategy/decisions stay in the kickoff call. |
| `delivery.md` | 8-week pilot delivery SOP — two parallel tracks (AI brain + workflow). Phase 0 pre-onboarding → Phase 1 context → Phase 2 data + baseline → Phase 3 workflow build → Phase 4 handoff. The 8 brain-category files. |
| `workflow-build.md` | Phase 3 build SOP — prereq check (APIs + brain + scope), n8n vs divinecore-v2 decision, 6-step build flow, where things commit. |
| `case-studies.md` | **Canonical record of Pang's shipped client work.** OuterSignal, Cracked.ai, Reading Rhythms, Reliable Medicare. Numbers + tech stacks + outcomes. Pull from here for proposal docs and any deeper credibility section; condensed versions in `pang.md` / `about_me.py` mirror it. |

## `offer.md` vs `guarantee.md`

These two files are **complementary**, not alternatives. Load both for any agent generating proposals or running sales conversations.

- **`offer.md`** is the **strategic offer**: pricing anchors, payment model, channel/audience map, pilot timeline, scope envelope, expansion path. Answers *what we sell and for how much*.
- **`guarantee.md`** is the **legal mechanic**: TYPE A vs TYPE B verbatim paragraphs, what triggers each, what doesn't trigger, buyer obligations, claim mechanic. Answers *what specifically we commit to and what voids it*.

The unified offer applies to every channel; only the **audience** and **naming discipline** differ by channel (see `offer.md` §08). The variant (TYPE A or TYPE B) of the guarantee depends on the specific JD or prospect (see `guarantee.md` §02).
