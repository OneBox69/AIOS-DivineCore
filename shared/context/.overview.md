# shared/context — Cross-Module Identity Context

The single source of truth for **who DivineSide is, who Mayank is, how we sound, who we're talking to, and where we're heading.** Any agent that produces text aimed at humans (sales emails, YouTube scripts, LinkedIn posts, DMs, follow-ups) MUST load the relevant files here before generating output.

## Why this folder exists

CLAUDE.md is for agents understanding the **system architecture** — modules, infrastructure, conventions. It is not voice-shaped. An agent reading CLAUDE.md to write a LinkedIn post would produce something that sounds like documentation.

`shared/context/` is the **voice-shaped** layer. Same facts, different framing — written so an LLM can lift identity, tone, positioning, and target audience directly into copy.

This is a deliberate split:
- **CLAUDE.md** — what the system IS (for agents that build/maintain the system)
- **shared/context/** — who the brand IS (for agents that speak as the brand)

## Loading discipline

- **Don't load every file every time.** Pick what's relevant for the task. A YouTube hook agent doesn't need `audience.md`'s full persona breakdown — only `voice.md` and `personal-info.md`.
- **Cite the source in the agent's system prompt.** Be explicit: "Use the tone described in `shared/context/voice.md`." This makes drift visible later.
- **When this content drifts from CLAUDE.md, update both.** They share facts; they shouldn't disagree.

## What does NOT go here

- **Architecture, infrastructure, module specs** — those stay in CLAUDE.md.
- **Knowledge bases / domain frameworks** — Kallaway lives in `branding-os/knowledge-base/` (Airtable-mirrored), Hormozi lives in `sales-os/knowledge-base/`. These are *expertise*, not *identity*.
- **Working examples / past content** — those live in Supabase + module folders. This is the identity layer, not the example library.
- **Personal data unrelated to brand voice** — health, finances, life logistics. Keep this folder narrow.
- **Secrets** — never. Reference only.

## Maintenance

- These files should evolve as the brand evolves. Treat them like a living style guide.
- When Mayank writes a piece of content that *feels* exactly right, mine it for `voice.md` updates.
- When a campaign launches or a positioning shifts, update `strategy.md` in the same week — agents will keep writing yesterday's pitch otherwise.
