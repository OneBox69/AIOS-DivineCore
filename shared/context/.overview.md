# shared/context — Cross-Module Identity Context

The voice-shaped layer of DivineSide. CLAUDE.md describes what the system **is** (architecture, modules, infrastructure). This folder describes who the brand **is** (identity, sales process, channel playbooks). Any agent producing text aimed at humans loads from here.

## Structure — three subfolders

```
shared/context/
├── identity/                    ← Who we are, how we sound, who we sell to, per-person personas
├── sales-and-delivery/          ← Offer, guarantee, sales process, delivery SOP
└── playbooks/                   ← Channel-specific tactical playbooks (LinkedIn, X, Upwork Loom)
```

Each subfolder has its own `.abstract.md` (one-liner) and `.overview.md` (file breakdown + loading discipline). Read those when scoping a task — they hold the detail.

| Subfolder | Holds | Detailed overview |
|---|---|---|
| **identity/** | Brand + persona layer | [identity/.overview.md](identity/.overview.md) |
| **sales-and-delivery/** | Deal flow + shipping | [sales-and-delivery/.overview.md](sales-and-delivery/.overview.md) |
| **playbooks/** | Channel-specific tactical content | [playbooks/.overview.md](playbooks/.overview.md) |

## Loading discipline — quick map by task

(Each subfolder's overview has the full version; this is the cross-folder cheat sheet.)

### Pang LinkedIn or X content
1. `identity/pang.md` — persona (who is writing)
2. `playbooks/linkedin-playbook.md` OR `playbooks/x-playbook.md` — channel structure
3. `identity/voice.md` — brand-level voice rules
4. (Optional) `playbooks/swipe-file.md` — raw examples

### Mayank video / YouTube content
1. `identity/mayank.md` — persona
2. `identity/voice.md`, `identity/strategy.md`
3. `branding_os/knowledge-base/` — Kallaway frameworks
4. (Optional) `playbooks/swipe-file.md` §03

### Cold-email outreach / sales reply qualifier
1. `identity/business-info.md`, `identity/voice.md`, `identity/audience.md`
2. `sales-and-delivery/sales-playbook.md` §01–02
3. The persona file for whoever the email is voiced as (`identity/mayank.md` or `identity/pang.md`)
4. `sales_os/knowledge-base/` — Hormozi frameworks

### Discovery call
1. `sales-and-delivery/sales-discovery-call.md` — live-call playbook
2. `sales-and-delivery/sales-playbook.md` — broader SOP
3. `sales-and-delivery/offer.md` — what we sell
4. `sales-and-delivery/delivery.md` — so the call doesn't promise what delivery can't ship
5. `identity/business-info.md`, `identity/audience.md`, `identity/voice.md`
6. Persona for whoever's running the call

### Upwork Loom recording / proposal generation
1. `playbooks/upwork-loom-script.md` — script structure
2. `sales-and-delivery/guarantee.md` — the guarantee terms
3. `identity/pang.md` — persona (Loom is Pang-voiced)

### Pilot delivery (after the deal closes)
1. `sales-and-delivery/delivery.md` — the 6-week SOP
2. `sales-and-delivery/offer.md` — so it knows what was promised
3. `sales-and-delivery/intake-form.md` — Phase 0 form content
4. `sales-and-delivery/workflow-build.md` — Phase 3 build SOP
5. `identity/voice.md` for any client-facing copy

### Outreach DMs (Networking OS)
1. `identity/business-info.md`, `identity/voice.md`, `identity/audience.md`
2. Platform-specific conventions (handled by the bot itself)

## What goes here vs. CLAUDE.md

| | Goes in CLAUDE.md | Goes in shared/context/ |
|---|---|---|
| System architecture | ✅ | — |
| Module specs | ✅ | — |
| Infrastructure config | ✅ | — |
| Repo conventions | ✅ | — |
| Brand identity, voice, audience | — | ✅ |
| Sales process, offer, guarantee | — | ✅ |
| Channel content tactics | — | ✅ |
| Per-person persona | — | ✅ |

When the same fact appears in both (e.g. Pang's role), keep them in sync. They share facts; they shouldn't disagree.

## What does NOT go here

- Architecture / module specs → CLAUDE.md
- Knowledge bases (Kallaway, Hormozi) → `branding_os/knowledge-base/`, `sales_os/knowledge-base/`
- Working examples / past content → Supabase + module folders
- Personal data unrelated to brand voice — keep this folder narrow
- Secrets — never. Reference only.

## Maintenance

- These files evolve with the brand. Treat like a living style guide.
- When a piece of content lands exactly right, mine it for the relevant `identity/` file.
- When positioning shifts, update `identity/strategy.md` within the week — agents will keep writing yesterday's pitch otherwise.
- When a new playbook or subfolder is added: per [CLAUDE.md §15](../../CLAUDE.md), create the `.abstract.md` and `.overview.md` in the same commit.
