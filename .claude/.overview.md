# .claude — Claude Code Project Config

Project-scoped Claude Code configuration. Anything here is version-controlled and applies to everyone working in this repo.

## Structure

| Path | Role |
|------|------|
| `commands/` | Custom slash commands. Each `*.md` file becomes an invocable `/<filename>` command. |

## Slash commands today

(See `commands/` directly — `.abstract.md` is intentionally absent there since Claude Code would register it as a slash command. See CLAUDE.md §15.)

| Command | Purpose |
|---------|---------|
| `/prime` | Load full DivineCore context at session start. Reads `CLAUDE.md` + `README.md`, lists `divinecore-v2/`, runs `git log --oneline -20`, and reads `.claude/session.md` if present. |
| `/resume` | Lightweight re-entry. Reads `CLAUDE.md` (and `.claude/session.md` if present) and gives a one-paragraph briefing. |
| `/save-context` | Writes a ≤300-word session summary to `.claude/session.md`. Optional — git log is the primary progress record (see commit `13faee3`). |

## Conventions

- **One command, one job.** Same rule as agents (CLAUDE.md §12).
- **Slash commands are version-controlled** — they're for the team, not personal preferences.
- **Don't put `.abstract.md` in `commands/`** — it gets registered as `/.abstract`. Folder is small enough to read directly.

## What does NOT go here

- Personal Claude Code settings (those go in `~/.claude/`, not the repo).
- Agent system prompts for product modules — those go in `<module>/agents/`.
- Long-form architecture context — that goes in `CLAUDE.md` at the repo root.
