# DivineCore — Repo Root

DivineSide's internal AI Operating System. The repo holds five business-function modules, a code-first v2 Python stack, and the infrastructure glue that ties everything to Discord and n8n.

## Top-level structure

| Path | Role | Status |
|------|------|--------|
| `CLAUDE.md` | Full company + architecture context for AI agents. Source of truth for everything not derivable from code | Live |
| `README.md` | Public-facing repo summary | Live |
| `agent.py` | Standalone OpenAI-based "client discovery agent" stub at the root | Skeleton (`main()` only prints) |
| `branding_os/` | Module 1 — Creative Intelligence (IMAGYN/LYRA/MONTAGE/FORGE/VIGIL) | IMAGYN active; rest scaffold |
| `sales_os/` | Module 2 — Revenue System (hosts the Upwork integration + `/upwork` web layer) | Upwork live; agent layer planning |

| `pulse/` | Module 4 — Alert + Awareness | **ACTIVE in production** (n8n) |

## Two parallel development tracks

1. **n8n-orchestrated stack** (current production): Discord → middleware → n8n workflows → agents → Airtable/Supabase. Everything Pulse-related runs here today.
2. **divinecore-v2 code-first runtime** (in build): Python services owned end-to-end in this repo. Will eventually replace n8n for agent execution.

The two coexist. Don't migrate working n8n workflows to v2 without a clear reason — the rule is "manual first, automate second, code-ify third."

## Conventions

- See CLAUDE.md §11–§15 for the full set of rules.
- **Tiered context (L0/L1/L2)**: every folder has `.abstract.md`, strategic folders also have `.overview.md`. See CLAUDE.md §15.
- **One agent, one job.** Never combine responsibilities.
- **Workflow exports go to `<module>/workflows/`** in the relevant module folder.
- **Secrets never in this repo.** `.env` references only.

## Where to start

- New session? Run `/prime` (loads CLAUDE.md + README + git log + this overview).
- Building a new module agent? Read CLAUDE.md §6 for that module, then look at the module folder's `.overview.md`.
- **Writing copy / content / outreach?** Start at [shared/context/.overview.md](shared/context/.overview.md) — every writing agent loads from there.
- Touching the Python runtime? Start at [divinecore-v2/.overview.md](divinecore-v2/.overview.md).
- Touching CI/CD? Start at [.github/workflows/.overview.md](.github/workflows/.overview.md).
